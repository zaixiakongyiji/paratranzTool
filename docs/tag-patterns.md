# 保存前检查 - 标签格式匹配维护手册

> 本文档用于长期维护翻译校验中的**标签格式匹配规则**。  
> 随着项目翻译推进，遇到新的标签格式时，请按照本文档指引更新规则。

## 概述

保存前检查功能在提交翻译时，会自动比对原文和译文中的标签是否一致。标签提取由正则表达式驱动，所有规则定义在：

📄 [`src/utils/validation.js`](../src/utils/validation.js) → `TAG_PATTERNS` 数组

---

## 当前已支持的标签格式

### 1. 尖括号标签 `<...>`

**正则**：`/<[^>\n]*>/g`

**匹配策略**：从 `<` 开始，到**同一行内最近的** `>` 结束，中间允许包含嵌套的 `<`。

| 格式类型 | 示例 | 匹配结果 |
|---------|------|---------|
| 简单变量 | `<player_name>` | `<player_name>` |
| 带参数函数 | `<npc_i(felicia)>` | `<npc_i(felicia)>` |
| 嵌套标签 | `<switch_privates(Your <adjective_penis1>` | `<switch_privates(Your <adjective_penis1>` (整体) |
| 多重嵌套 | `<outer <inner <deep>` | `<outer <inner <deep>` (整体) |

**已确认的实际标签样本**：

```
<adjective_penis1>         — 属性描述变量
<penis1>                   — 身体部位变量
<adjective_vagina1>        — 属性描述变量
<vagina1>                  — 身体部位变量
<npc_i(felicia)>           — NPC 引用（带参数）
<switch_privates(...)>     — 条件分支标签（跨多行，内部含嵌套标签）
```

## 如何添加新的标签格式

### 步骤 1：识别新格式

在翻译过程中遇到保存前检查**未检出**的标签格式时，记录下原始文本和标签样本。

常见的潜在新格式（尚未遇到，待确认）：

| 格式 | 示例 | 备注 |
|------|------|------|
| 方括号变量 | `[player_name]` | 部分游戏引擎使用 |
| 花括号占位符 | `{0}`, `{count}` | 通用格式化字符串 |
| 百分号格式化 | `%s`, `%d`, `%1$s` | C 风格格式化 |
| 双花括号模板 | `{{variable}}` | 模板引擎语法 |
| 美元符变量 | `$variable`, `${name}` | 脚本语言风格 |

### 步骤 2：编写正则

在 `TAG_PATTERNS` 数组中添加新的正则表达式：

```js
// src/utils/validation.js
const TAG_PATTERNS = [
  // 已有：尖括号标签
  /<[^>\n]*>/g,
  
  // 示例：如果需要支持方括号变量 [variable_name]
  // /\[[a-zA-Z_]\w*\]/g,
  
  // 示例：如果需要支持花括号占位符 {0}, {name}
  // /\{[^}\n]*\}/g,
];
```

**编写正则的注意事项**：

1. **必须使用 `g` 全局标志** — `extractTags` 函数依赖全局匹配
2. **建议排除换行符 `\n`** — 避免跨行匹配导致误判
3. **尽量精确** — 避免匹配到正常的翻译文本
4. **注意互斥** — 新正则不应与已有正则产生重叠匹配

### 步骤 3：补充测试用例

在 `src/utils/validation.test.js` 中添加对应的测试：

```js
it('提取新格式标签', () => {
  const text = '包含 [new_tag] 的文本';
  const tags = extractTags(text);
  expect(tags).toContain('[new_tag]');
});
```

### 步骤 4：运行测试

```bash
npx vitest run src/utils/validation.test.js
```

### 步骤 5：更新本文档

在上方「当前已支持的标签格式」章节中添加新格式的说明。

## 校验规则一览

除标签匹配外，保存前检查还包含以下校验项：

| 检查项 | 级别 | 说明 |
|--------|------|------|
| 行数一致性 | ⚠️ 警告 | 原文与译文的换行数不同 |
| 缺失的标签 | ⚠️ 警告 | 原文中存在但译文中缺失的标签 |
| 多余的标签 | ⚠️ 警告 | 译文中存在但原文中没有的标签 |
| 首尾空白符 | ⚠️ 警告 | 前导/末尾的换行或空格不一致 |

所有校验结果均为**警告级别**，不阻断保存。用户可以在弹窗中点击「继续」强制提交。

如需添加新的校验规则，在 `validation.js` 中编写对应的 `check*` 函数，并在 `validateTranslation` 中调用即可。
